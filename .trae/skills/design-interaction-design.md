# 交互设计工作流

你现在要完成一项全面的交互设计任务。你已安装以下 Skill，请按步骤串联使用：

## 步骤 1：用户研究与画像构建（获取层）
使用 **Ux Researcher Designer** 完成：
- 基于数据驱动方法生成用户画像（Persona）
- 绘制用户旅程地图（Journey Map），标注关键触点和情绪曲线
- 设计可用性测试框架和测试任务
- 识别用户痛点、需求和行为模式
- 产出用户研究洞察报告

输出用户画像文档、旅程地图和研究洞察。

## 步骤 2：信息架构与交互方案设计（分析层）
使用 **UI/UX Pro Max** 完成：
- 设计信息架构（IA），定义内容层级和导航结构
- 制定视觉风格方向（色彩/字体/间距/图标风格）
- 设计组件规格和交互状态（默认/悬停/按下/禁用/加载）
- 编写界面文案和微文案（按钮/提示/错误信息）
- 构建设计系统令牌和组件文档

输出信息架构图、交互规格和设计系统文档。

## 步骤 3：UX 可用性审计（分析层）
使用 **UI Audit** 完成：
- 基于 UX 成熟原则自动审计界面设计
- 评估视觉层级是否清晰（主次关系、视觉引导）
- 检查认知负荷是否合理（信息密度、决策复杂度）
- 审查导航设计（可发现性、一致性、回退路径）
- 评估无障碍性基础指标

输出 UX 审计报告和优化建议清单。

## 步骤 4：微交互与动效设计（分析层）
使用 **UI/UX Design Guide** 完成：
- 设计移动优先的交互模式和手势方案
- 定义微交互（按钮反馈、加载动画、状态切换）
- 制定色彩系统和字体排版层级
- 确保设计符合 WCAG 2.2 无障碍标准
- 集成 Tailwind + Shadcn/ui 的交互组件规范

输出微交互规范和动效设计方案。

## 步骤 5：交互原型优化与实现（输出层）
使用 **human-optimized-frontend** 完成：
- 通过量化评估共同优化美学、动效和用户体验
- 确保交互动效流畅自然（缓动曲线、时序控制）
- 优化界面视觉细节（阴影、渐变、过渡）
- 生成视觉美观、体验良好的前端界面代码
- 量化评分指导迭代优化

输出优化后的前端交互代码和评估报告。

## 步骤 6：无障碍合规实现（输出层）
使用 **Accessibility** 完成：
- 实现语义化 HTML 结构和适当的 ARIA 标签
- 确保键盘导航完整（Tab 顺序、焦点管理、快捷键）
- 验证颜色对比度（文本 4.5:1，大文本 3:1）
- 测试屏幕阅读器兼容性
- 实现实时区域（Live Region）和表单标签

输出 WCAG 2.1 AA 合规的前端代码。

## 最终输出
将以上步骤的结果整合为完整的交互设计包，交付以下文件：
1. **用户研究报告**：用户画像、旅程地图、痛点洞察
2. **信息架构与交互规格**：IA 图、组件规格、状态定义
3. **UX 审计报告**：视觉层级/认知负荷/导航评估和优化建议
4. **微交互与动效方案**：交互模式、动效规范、时序定义
5. **前端交互代码**：量化优化后的高质量界面实现
6. **无障碍合规报告**：WCAG 2.1 AA 检查结果和合规代码

## Skills
- ux-researcher-designer: Ux Researcher Designer - UX research and design toolkit for Senior UX Designer/Researcher including data-driven persona generation, journey mapping, usability testing frameworks, and...
- ui-ux-pro-max: UI/UX Pro Max - UI/UX design intelligence and implementation guidance for building polished interfaces. Use when the user asks for UI design, UX flows, information architecture, visual style direction, design systems/tokens, component specs, copy/microcopy, accessibility, or to generate/critique/refine frontend UI (HTML/CSS/JS, React, Next.js, Vue, Svelte, Tailwind). Includes workflows for (1) generating new UI layouts and styling, (2) improving existing UI/UX, (3) producing design-system tokens and component guidelines, and (4) turning UX recommendations into concrete code changes.
- ui-audit: UI Audit - AI skill for automated UI audits. Evaluate interfaces against proven UX principles for visual hierarchy, accessibility, cognitive load, navigation, and more. Based on Making UX Decisions by Tommy Geoco.
- ui-ux-design: UI/UX Design Guide - Expert guidance on mobile-first UI/UX design, color systems, typography, accessibility (WCAG 2.2), Tailwind + Shadcn/ui integration, micro-interactions, and...
- human-optimized-frontend: human-optimized-frontend - Generates a frontend interface that is visually pleasing and experientially sound by jointly optimizing aesthetics, motion graphics, and user experience (UX) using quantified evaluation. Use only when the user explicitly invokes this skill by name to redesign a frontend. Trigger keywords: use human-optimized-frontend, redesign frontend, redesign interface.
- accessibility: Accessibility - Build WCAG 2.1 AA compliant websites with semantic HTML, proper ARIA, focus management, and screen reader support. Includes color contrast (4.5:1 text), keyboard navigation, form labels, and live regions.

Use when implementing accessible interfaces, fixing screen reader issues, keyboard navigation, or troubleshooting "focus outline missing", "aria-label required", "insufficient contrast".
